# 투자자 구분 번호 범례 (종목투자자별 매매 필드)

`/stock/m001/invest_hist`, `/stock/m001/invest_info_port` 등 투자자별 매매 endpoint의
응답은 투자자를 **이름이 아니라 번호(_01 ~ _20)** 로 준다. 원 명세(checkapi-specs.json)에는
이 번호→이름 범례가 없어(명세엔 "8=기관"만 표기), **삼성전자(005930) 2025-06-02 실데이터의
산술 구조로 검증**해 아래 표를 확정했다.

## 필드 패밀리 (접미사 `_01`~`_20` = 투자자 번호)

| F-code | 의미 |
| --- | --- |
| F06505_NN | 종목투자자별 **매도거래량** |
| F06507_NN | 종목투자자별 **매수거래량** |
| F06508_NN | 종목투자자별 **순매수거래량** |
| F06509_NN | 종목투자자별 **매도거래대금** |
| F06510_NN | 종목투자자별 **매수거래대금** |
| F06511_NN | 종목투자자별 **순매수거래대금** |

## 투자자 번호(NN) 범례

| NN | 투자자 | 확신도 |
| --- | --- | --- |
| 01 | 금융투자 | 표준순서 |
| 02 | 보험 | 표준순서 |
| 03 | 투신 | 표준순서 |
| 04 | 사모 | 표준순서 |
| 05 | 은행 | 표준순서 |
| 06 | 기타금융 | 표준순서 |
| 07 | 연기금 등 | 표준순서 |
| **08** | **기관계** | **확정** (명세 표기 + 산술: 08 = 합(01..07) + 13) |
| 09 | 기타법인 | 높음 (파티션 + 규모/표준순서) |
| 10 | 개인 | 높음 (파티션 + 규모/표준순서) |
| **11** | **외국인** | **높음** (아래 파티션 검증) |
| **12** | **전체(합계)** | **확정** (= 총거래량, 순매수 = 0) |
| 13 | 국가/지자체 | 확정 (산술: 기관계 구성요소) |
| 14~16 | 외국인/기타 보조 집계 | **불확정** — 필요 시 실호출로 확인 |
| 17~20 | 미사용(0) | 확정 |

### 검증 근거 (2025-06-02, 005930, 순매수거래량 F06508)
- `08(기관계) = 합(01..07) + 13(국가/지자체)` — 매수·매도·순매수 3패밀리 모두 성립.
- `12 = 총거래량`(12,870,515), 순매수 `12 = 0`.
- `08 + 09 + 10 + 11 = 0` (순매수 합) → 최상위 파티션이 **기관/기타법인/개인/외국인** 임이 확정.
  실측: 기관 -600,990 + 기타법인 -24,557 + 개인 -237,061 + 외국인 +862,608 = 0.

## F-code 패밀리 (매도/매수/순매수) — `F06506`은 없다

| F-code | 의미 |
| --- | --- |
| `F06505_NN` | 매도거래량 |
| **`F06507_NN`** | **매수거래량** (F06506 아님 — **F06506은 존재하지 않는다**) |
| `F06508_NN` | 순매수거래량 |
| `F06509` / `F06510` / `F06511` | 매도 / 매수 / 순매수 **거래대금** |

`data_list` 는 없는 F-code를 조용히 버리므로 `F06506_12`를 넣어도 오류 없이 필드만 사라진다.
**요청한 개수와 반환된 개수를 대조할 것.**
"그 종목이 그날 거래됐는가"는 **`F06505_12` 또는 `F06507_12` > 0** (전체=12)으로 판정한다.

## 자주 쓰는 조회 지름길

- **외국인 순매수(거래량)** = `F06508_11`, **외국인 순매수(대금)** = `F06511_11`
- **기관 순매수** = `F06508_08`, **개인 순매수** = `F06508_10`
- 종목별로 뽑으려면 `/stock/m001/invest_hist jcode=<코드> sdate=.. edate=..` 호출 후
  `F06508_11`(외국인) 등 필요한 번호만 보면 된다. `data_list=F16013,F12506,F06508_08,F06508_10,F06508_11`
  처럼 필요한 F-code만 넘기면 응답이 가벼워진다.

## 지수/업종 레벨(m002·m004 invest)은 번호가 다르다 (실측 확인)
위 표는 **개별종목(m001)** 기준이다. **지수·업종 레벨(m002 코스피 / m004 코스닥)의
`invest_hist_info`는 투자자 번호 체계가 다르다.** 2026-07-03 KRX/코스닥 리포트로 확정:
- 금융투자=**1**, 투신=**3**, **연기금=6**(m001의 6=기타금융과 다름!), 기관계=8, 개인=10, 외국인=11.
- 예: 코스닥 연기금 순매수 = `F06511_06`. (m001식 code 7로 뽑으면 0이 나옴 — 주의)
- 외국인/기관/개인(11/8/10)은 두 레벨 공통.

## 주의
- 14~16번의 정확한 정의는 미확정이다. 외국인 세부(외국인/기타외국인) 구분이 필요하면
  실데이터로 파티션을 다시 맞춰 확인할 것.
- 이 범례는 stock 투자자별 매매(F0650x/F0651x) 계열 기준이다. 다른 계열(선물/옵션 투자자 등)은
  번호 체계가 다를 수 있으니 그 endpoint의 데이터로 별도 검증한다.
