# 시황분석 치트시트 (마켓 코멘터리용)

35개 시황 시나리오를 실제로 검색·호출 테스트해 정리한 표다. "무엇을 보고 싶다"에서
바로 endpoint로 가기 위한 지도.

## ★ 거래소 패밀리 — KRX / NXT / 통합 (실측 확정, 반드시 먼저 볼 것)
2025년 넥스트레이드(NXT) 출범으로 같은 종목이 **KRX·NXT 두 곳**에서 거래된다. stock 패밀리는
거래소×시장×레벨로 나뉜다:

| 레벨 | KRX | NXT | 통합(KRX+NXT) |
| --- | --- | --- | --- |
| 종목(코스피) | m001 | m222 | **m224** |
| 종목(코스닥) | m003 | m223 | **m225** |
| 지수·업종(코스피) | m002 | m226 | **m228** |
| 지수·업종(코스닥) | m004 | m227 | **m229** |

- **거래대금/거래량/순위(rank)는 KRX-only면 실제보다 30~45% 과소집계.** 실측(2026-07-03): 삼성전자
  KRX 94,548억 vs **통합 171,835억**(NXT ≈45%); 코스피 지수 거래대금 KRX 45.5조 vs **통합 71.9조**.
  → **시장활동(거래대금·거래량·순위·장중) 기본은 통합**. `market_snapshot(venue="unified")`이 기본.
- **통합 rank**(m224/m225)는 `up_code` 선택(KRX는 필수), KRX와 동일 종목수·필드. **통합 지수 거래대금**은
  m228/m229 `hist_info`(F15023/F15015만; **지수 값·등락률 F15001/F15004는 0** → 지수레벨/등락은 KRX m002/m004).
- **종가·등락률 "공식값"은 KRX**(통합/NXT는 실질 마지막체결 = NXT 연장거래 포함해 다름: 삼성 KRX +8.22% vs 통합 +9.97%).
- **통합에 없어 KRX 유지 불가피**: `hist_info_port`(복수종목 일봉)·`invest_hist`(종목별 수급)·`short/loan/credit`
  (공매도·대차·신용)·`new_high`·`mass`. → 기간(복수종목) 스크리너·공매도·수급·신고가는 KRX. `short_daily`는 원래 KRX가 정답.
- 호가: 세 거래소 모두 `hoga_info(_port)` 제공(46필드 10단계). 실시간 적재는 `hoga_ingest.py`/[[hoga-ingest-setup]].
- ⚠ spec 메뉴에 NXT/통합이 `state=off`로 표기돼 있으나 **실측상 모두 데이터 반환**(낡은 메타데이터).
- **그 외 stock 패밀리**(KRX/NXT/통합 외): 수익증권 `m008`·신주인수권 `m009`·OTCBB `m010`·
  KONEX 종목 `m118`/업종 `m121`·섹터지수 `m167`(KRX100/KRX300/섹터·테마)·국내기타지수 `m168`(파생/전략지수).
  전체 6개 도메인 시장명↔코드 룩업은 리포 `docs/checkapi/market-index.md` 참조.

## 순위(rank) 계열: criteria_code 정렬코드

`rank` 계열은 `up_code`(업종코드)와 `criteria_code`(정렬 기준 F-code)가 **둘 다 필수**다.
정렬코드는 endpoint마다 다르다 — 전체 목록은 CHECK Data-Set에 있고, 명세에서 확인된 상용값:

- **`/stock/m001/rank`**: 거래량 `F15015` · 거래대금 `F15023` · 시가총액 `F15028` · 등락률 `F15004` · 주가 `F15001`
- **`/stock/m001/rank_invest`**: `F06508_NN` (투자자 순매수거래량; 외국인=_11, 기관=_08, 개인=_10)
- **`/stock/m001/rank_short_date`**: 공매도수량 `F33094` · 공매도금액 `F33095`

`up_code`: **시장 전체는 `1` 로 고정**(실측: 2,467종목 반환). 특정 업종만 볼 때는 업종코드가
필요한데 전용 목록 endpoint가 없어, `upjong_info`(소속업종정보) 응답의 업종코드를 참고하거나
Data-Set을 확인한다.

## basic_info_all = 개별종목 만능 카드

`/stock/m001/basic_info_all` 한 endpoint(150필드)에 개별종목 시황 지표가 대부분 들어있다:
외국인보유율/보유주식수, 상한가·하한가, 시가총액, 상장주식수, 각종 대비/등락 등.
"이 종목의 X" 류 질문은 먼저 여기서 F-code를 찾아본다(`get_endpoint_spec.py`로 필드 확인).
※ 단, "X한 종목들의 리스트/순위"는 basic_info가 아니라 `rank`로 푼다.

## 검색이 basic_info류만 뱉을 때
endpoint 제목이 전부 일반명(`기본정보`/`일별정보`)이라, 지표가 **응답 필드에** 있으면
검색이 약할 수 있다. 이럴 때:
1. `get_endpoint_spec.py`로 후보의 필드 목록을 열어 원하는 지표 F-code가 있는지 확인,
2. "상위/순위"면 `rank`+`criteria_code`,
3. "개별종목 종합"이면 `basic_info_all` 을 우선 검토.

## "없음"을 말하기 전에 — 제목검색 실패 ≠ 범위 밖
endpoint 제목은 일반명뿐이라, 많은 지표는 **제목엔 없고 응답 필드에만** 있다.
`search_endpoints`(제목 위주)가 비어도 `search_fields.py <지표>`로 필드를 확인하기 전엔
"없다"고 하지 말 것. 아래는 명세 전수 감사 결과다.

### 실은 있다 (필드에만 존재 — search_fields로 찾을 것)
`search_endpoints`로는 잘 안 잡히지만 필드엔 있는 대표 지표:
- **베이시스**(시장/이론, `F04001`/`F04002`, future/m005), **괴리율**, **차익/비차익**(프로그램)
- **신용융자 잔고**, **투자자 과열/과열종목**, **액면**, **상장폐지 관련**
→ 이런 건 범위 밖이 아니다. `search_fields.py`로 endpoint·F-code를 찾아 답한다.

### 진짜 범위 밖 (제목·필드 모두 없음 — 확인됨)
- ~~환율/FX~~ **정정: 원/달러는 됨** — `bond/m023 basic_info 00USDSP`(당일 spot·기준율, 실측 1,525.6).

- **국내주식 배당·PER/PBR** — *일별 시세 필드로는* 없음. **단 컨센서스(`etc/cons`)가 국내종목
  PER(382100)·PBR(382500)·배당수익률(432400) 등을 실측 제공**(comp-fundamentals-analysis.md 참조).

### 장중(intraday) 조회 — intra_date
- **`/stock/m002(또는 m004)/intra_date jcode=<코드> edate=<날짜>`** → 과거일 1분봉 조회 가능.
  시각 F20004_02(HMMSSss, 예 9010000=09:01), 종가 F20008_02, 등락률 F20041_02.
- 원하는 시각 이상 첫 행을 고르면 그 시점 스냅샷. **KRX300(m167)은 intra_date 없음**(장중 미지원).
- 장중 투자자 수급은 `invest_intra_date`(누적 '수량' 기준, F23234_NN 등) — 억원 대금은 별도 산출 필요.
- **예상시가·괴리율**: 동시호가 선물 예상체결 기반 산식이 필요(시황팀 공식) — CHECK 데이터로 직접 재현 안 됨.

## 지수·업종 레벨 = m002(코스피)/m004(코스닥) — "지수 값" 됩니다
(중요 정정: 지수 값은 제목검색에 안 잡혀 한때 "범위 밖"으로 오판했으나, **실호출로 조회 가능 확인**.)
- **m002 = 코스피 업종/지수, m004 = 코스닥 업종/지수.** `jcode`에 **업종코드**(지수코드)를 넣는다.
- **지수 코드 목록**: `/stock/m002/code_info`(코스피) · `/stock/m004/code_info`(코스닥). F16013=코드, F16002=한글명.
  - 실측: **코스피 종합 = `1`**, KOSPI200 = `51`, KOSPI100 = `61` …
- **지수 값·등락률·거래대금**: `/stock/m002/hist_info jcode=<코드> sdate= edate=`
  (F15001=현재가(지수), F15004=등락율, F15023=거래대금). 실측: KOSPI 2023-04-12 = 2550.64(+0.11%).
- **시장 단위 투자자 수급**: `/stock/m002/invest_hist_info jcode=<코드>` — 순매수거래대금 `F06511_11`(외국인)/`_08`(기관)/`_10`(개인). 억원 = 값/1e8.
  ※ 집계기준(정규장/전체·잠정/확정)에 따라 공표 시황과 ±수% 차이 날 수 있음.
- 과거 데이터: hist 계열은 `sdate/edate`로 과거 조회 가능(2023년 확인).

### 국고채 지표금리 (일별) — `bond/m058/jipyo_list` + `bond/m038/hist_info`
지표물이 로테이션되지만 **`jipyo_list`가 `edate`를 받아 그날의 지표물을 준다** → 과거일도 정확.
1. `bond/m058/jipyo_list edate=<날짜>` → 만기별(F14131=2·3·5·10·20·30·50년) 지표물 ISIN(F16013).
   ※ 만기10년엔 물가채(물가…)와 명목국고(국고…) 둘이 있으니 **F16003 이름이 "국고…"인 명목물** 선택.
2. `bond/m038/hist_info jcode=<ISIN> sdate=edate=<날짜>` → **F15175=수익률** = 국고채 지표금리.
- 실측: 3년 07-03 = 3.745(≈시황 3.752, 07-02=3.747 정확), 10년 = 4.168(≈4.198).

### 코드 카탈로그 검색 — `scripts/search_codes.py <키워드>` (필수 습관)
지수·지표물·FX·해외금리·경제지표·종목은 **명세에 없고 코드로만** 존재해 `search_endpoints`/`search_fields`로
안 잡힌다. `cache_codes.py`로 code_info/checkcode 카탈로그(35,340개)를 1회 캐시한 뒤 `search_codes.py`로 검색.
예: `원달러`→00USDSP, `WTI`→USCOMCL1D, `달러지수`→USDXYD, `미국 10Y`→GBUS10Y, `삼성전자`→005930.
→ "없음" 판정 전 **반드시 여기까지** 확인(과거 지수·원달러·해외금리·WTI를 이 단계 누락으로 오판).

### 단말기/Excel 매뉴얼 ≠ CHECK-API (코드체계가 다름)
CHECK-API의 자기 카탈로그(checkcode/code_info)를 기준으로 판단할 것. **Excel 프리미엄서비스(견적서 별도 상품)에
보여도 API엔 없을 수 있다.** 실측: Excel의 '주요원유가격-Brent(현물) 144743'(숫자코드+CH/CT 접두어)은 checkcode(전부
알파벳코드)에 없고, 그 코드로 `etc/economic/indicator` 호출 시 502. → **Brent/두바이 현물은 API 부재**(원유가격은 WTI선물 USCOMCL1D만).
- 각국 국채/기준금리·환율·PMI·생산 등. ※ 한국 국고채·원달러는 **월별(M)만** → 일별은 채권 지표물/외환 경로.
- ※ 반환이 오래된 것부터일 수 있으니 최신은 **max(TIME)** 으로.

### 장마감 시황 재현 검증 (2026-07-03, KRX Brief 대조)
