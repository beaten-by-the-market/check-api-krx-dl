# 장중(1분봉) flash crash / 급등 탐지·원인분석

히스토리컬 1분봉으로 **직전 봉 대비 순간 급락(flash crash)·급등(flash spike) 후 되돌림**을
탐지하고, 거래량 급증·지수 동반·당일 공시로 원인을 규명한다. 헬퍼 `scripts/flash_lab.py`
(등록 IP·샌드박스 밖). 아래는 전부 실호출 검증(2026-07).

```python
import flash_lab as fl
fl.flash_scan_day("20260703", universe_top=40, thresh=4.0)   # 특정일 시장 스캔 → flash 종목
fl.flash_moves("034020", "20260703")                          # 한 종목·하루 이벤트 목록
fl.explain_flash("002990", "20260703")                        # 최대 이벤트 원인분석
fl.flash_hunt("20260629","20260703", codes=[...], wick=6.0)   # 기간 헌트(일봉 wick→장중 확인)
# CLI:  python flash_lab.py scan 20260703  /  python flash_lab.py one 034020 20260703
```

## 검증된 데이터 구조 (실측)
- **히스토리컬 장중 = `intra_date`(1분봉)**: `jcode + edate(과거일 가능)`. 하루 ≈ 382봉
  (09:00 개장동시호가 ~ 15:30 종가). 시간 `F20004_02`=HHMMSSss(9010000=09:01), OHLC `F20005~08_02`,
  분거래량 `F20010_02`, 분거래대금 `F20011_02`.
- **`tick_info`(체결)·`intra_info`(10초봉)은 당일(실시간)만** → 과거 조회 불가. 초단위 히스토리 없음.
- **지수 1분봉**: `/stock/m002/intra_date jcode=1`(코스피 종합) · `/stock/m004/intra_date jcode=2`(코스닥)
  → 종가 `F20008_02`(문자열/DECIMAL). flash 시점 시장 동반 여부 판정.
- **시장 라우팅**: KOSPI=m001, KOSDAQ=m003. **m001에 코스닥 코드 → 에러(빈 결과 아님)** → fam 힌트/폴백.
- **일봉 OHLC**(wick 사전필터): 시가 `F15009`·고가 `F15010`·저가 `F15011`·종가 `F15001`.

## 거래소별 패밀리 — KRX / NXT / 통합 (정정: NXT 존재함)
메뉴 트리 실측 결과 stock 패밀리는 거래소×시장으로 나뉜다:

| 패밀리 | 거래소 | 시장 | 메뉴명 |
| --- | --- | --- | --- |
| m001 / m003 | KRX(거래소) | 코스피 / 코스닥 | 거래소 종목 / 코스닥 종목 |
| **m222 / m223** | **NXT(넥스트레이드)** | 코스피 / 코스닥 | NXT-KSP 종목 / NXT-KSQ 종목 |
| **m224 / m225** | **통합(KRX+NXT)** | 코스피 / 코스닥 | 통합-KSP 종목 / 통합-KSQ 종목 |
| m226~m229 | NXT / 통합 | 코스피 / 코스닥 | (업종·투자자) |

NXT/통합 패밀리도 `hoga_info`·`hoga_info_port`·`intra_date` 등 **동일 endpoint 세트**를 갖는다
(호가 46필드=10단계 매수/매도 호가+잔량+총잔량). 즉 **NXT 호가·통합(NBBO성) 호가가 API에 존재**한다.
**실측 확인(2026-07, 휴장일)**: `hoga_info_port`로 m001/m003(KRX)·m222/m223(NXT)·m224/m225(통합) **모두
데이터 반환**(휴장엔 마지막 스냅샷). spec 메뉴의 `state=off`는 낡은 메타데이터였고 실제로는 작동한다.
거래소 간 괴리도 관측됨 — 삼성전자 KRX 매수 309,500/매도 310,000 vs **NXT 314,500/315,000**(≈1.6%),
통합은 최선호가(NXT측) 반영. flash 탐지 기본은 KRX(m001/m003)이나 NXT/통합으로 바꾸면 같은 종목의
**거래소 간 가격 괴리·유동성 차이**를 볼 수 있다. (이전 "NXT 없음" 서술은 오류 — rank의 조건부
언급만 보고 성급히 판단; 근거가 와서 정정함.) 실시간 호가 적재는 `references/hoga-ingest-setup.md`.

## 탐지 로직
- **flash crash** = 분봉 저가가 직전 봉 종가 대비 `-thresh%` 이상 급락 → 이후 `rev_win`분 내
  `rev_frac` 이상 반등. **flash spike** = 고가 `+thresh%` 급등 → 되돌림. (직전 봉 = "직전 체결가" 근사)
- **개장 첫 `skip_open`봉·종가 `skip_close`봉 제외**(동시호가 구조적 변동).
- **되돌림(recover%)로 성격 분류**: ≥70% `V자(일시적)` = 대량주문/유동성 공백 · 30~70% 부분 ·
  <30% `지속(재료성)` = 공시/시장 방향.
- 기본 `thresh=4.0`. flash는 희소 → 잔잔한 날은 거의 안 잡히는 게 정상. 변동성 큰 날/종목에서 잡힘.

## 원인 분석 태그 (`explain_flash`)
1. **거래량 급증** = 그 분 거래량 / 당일 분거래량 중앙값 (≥5x면 태그).
2. **시장동반 vs 개별이슈** = 같은 분 지수 등락(±0.3% 임계). 지수 멀쩡한데 혼자 급변 = 개별.
3. **당일 공시** = `/news/gongsi/gongsi_jong jcode sdate=edate=date` 붙임.
4. **되돌림 형태** = V자/부분/지속.

**실측 대비(2026-07-03, 같은 -4%대 급락이 성격 정반대):**
- 두산에너빌리티(034020) 09:04 **-4.4%, 되돌림 73%** → 지수 +0.22%(개별)·거래량 6.7x → *V자 일시적(대량매도 한 방)*.
- 금호건설(002990) 09:17 **-4.9%, 되돌림 9%** → 지수 -0.40%(시장동반)·거래량 13.8x·당일공시(공매도 과열종목 연장) → *지속(재료성)*.

## 기간 헌트 (효율)
전 종목·전일 장중을 다 훑으면 비싸다 → **일봉 wick 사전필터**로 후보를 싸게 좁힌 뒤 장중 확정:
`daily_wick`의 아래꼬리 `(min(o,c)-low)/c`(crash 흔적)·위꼬리 `(high-max(o,c))/c`(spike 흔적)가
임계 이상인 (종목,날짜)만 `intra_date`로 확인. 실측: 금호건설 7/2 아래꼬리 13.2%→장중 -7.0%(14:01) 발굴.

## 함정
- **1분봉이라 초단위 급변은 못 본다**(과거 틱 미제공). "히스토리컬 최소 단위 = 1분".
- **거래량 배수는 당일 분거래량 중앙값 대비** 상대 신호(개장 물량이 커 중앙값이 낮음 → 배수 과대 경향).
- 잔잔한 날 thresh를 너무 낮추면(≤2.5%) 개장 변동·정상 노이즈가 섞인다.
- m001에 코스닥 코드 넣으면 예외 → `flash_scan_day`는 스냅샷 fam 힌트 사용, 단일 조회는 폴백.
