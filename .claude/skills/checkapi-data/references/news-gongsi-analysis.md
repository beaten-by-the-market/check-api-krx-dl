# 공시·뉴스 데이터 분석 치트시트 (news/gongsi + 시세 결합)

공시(`/news/gongsi/*`)·뉴스(`/news/news/*`) 피드를 시세·수급·공매도·대차·순위와 결합해
분석하는 시나리오 지도. **모든 시나리오는 실호출로 테스트 완료**(2026-06~07 데이터).
헬퍼는 `scripts/news_gongsi_lab.py`.

## 피드 구조 (실측)

### 공시(gongsi) — 하루 475~856건, KRX 공시·시장조치 스트림 (DART 원문 아님)
| endpoint | 용도 | 필수 파라미터 |
| --- | --- | --- |
| `/news/gongsi/gongsi_basic` | 하루 전체 공시 제목 | sdate, edate, (dcnt) |
| `/news/gongsi/gongsi_jong` | 종목별 공시 | jcode, sdate, edate |
| `/news/gongsi/gongsi_body` | 공시 본문(HTML 표) | ndate, ncode |

5개 축: **기업공시**(유증·CB·배당·자기주식·최대주주변경·합병분할·실적잠정·공정/자율공시·타법인양수)
· **시장경보/조치**(투자주의→경고→위험, 단기과열, 공매도과열, 소수계좌집중, 불성실공시, 조회공시)
· **매매/상장상태**(매매정지·상장폐지·관리종목·정리매매·신규/추가/변경상장)
· **파생신호**(주식선물·옵션 가격제한폭 확대요건 도달) · **통계·ETF**(회전율/변동성 상위, ETF LP).

### 뉴스(news) — 하루 8,500~12,000건, 언론사+시황 스트림
| endpoint | 용도 | 필수 파라미터 |
| --- | --- | --- |
| `/news/news/news_basic` | 하루 전체 뉴스 제목 | sdate, edate, (dcnt) |
| `/news/news/news_jong` | 종목별 뉴스 | jcode, sdate, edate |
| `/news/news/news_body` | 뉴스 본문 | ndate, ncode |
| `/news/news/news_mtvcd` | 뉴스원(언론사) 코드표 | - |

## 결합 키 & 함정 (반드시 지킬 것)
- **조인 키**: 공시 `종목코드(NCD)` = 시세 `jcode` ; `DATE` = 시세 입회일 `F12506` = `sdate/edate`.
- **뉴스엔 종목태그(NCD)가 없다.** 종목별 뉴스는 `news_jong(jcode)`로만. (공시 basic엔 NCD 있음)
- **시장 라우팅**: KOSPI = `stock/m001`, KOSDAQ = `stock/m003`. 종목코드만으론 구분 불가 →
  `pick_fam()`이 m001 조회 후 비면 m003 폴백. **이걸 놓치면 코스닥 종목이 전부 빈 결과**로 나온다.
  힌트: 공시 `MTVCD` = 300 거래소공시(KOSPI)·320 코스닥공시·310 선물옵션·330 코넥스.
- **대형주 뉴스 502**: 삼성전자 한 달 뉴스는 5,000건+ → 한 번에 조회 시 프록시 502.
  `news_stock_chunked()`로 8일 단위 분할·합산. (`call()`엔 502/타임아웃 3회 재시도 내장)
- **공시 발표 시각**: 시장경보류(투자주의·단기과열·공매도과열)는 대개 **장마감 후 20:00경 일괄**.
  이벤트 D0는 "이벤트일 이상 첫 거래일"로 잡아야 정확(헬퍼가 처리).
- **정정 공시**: `(정정)` 접두사 필터링 안 하면 이벤트 중복 카운트.
- **뉴스원(MTVCD)**: 0 시장조치 · 200~299 언론사 · 261 체크시황 · 297/298 인포스탁 · 500 CHECK시황.

### P. 시세→공시 역방향 (price-first: 스크리닝 → "왜 움직였나")
시세에서 시작해 **변동폭·거래대금 급변·주목 시황** 종목을 먼저 골라내고 공시를 붙인다.
러너: `scripts/scenarios_price_first.py` (PS1~PS7). 전부 실측 검증됨.
| # | 스크리닝 | 결합 | 실측(2026-06) |
| --- | --- | --- | --- |
| PS1 | 당일 등락률 상·하위 | rank+gongsi_stock | 콘텐트리중앙 −28.9% → **회생절차개시+매매정지+풍문해명** |
| PS2 | 당일 거래대금 상위 | rank+gongsi_stock | SK하이닉스 18조 → 중대재해+장래계획, 삼성전기 공급계약 |
| PS3 | 기간 수익률 상·하위 | hist+gongsi_stock | 금호건설 +146.8% → 공급계약·투자경고 체인 |
| PS4 | 거래대금 급증(최근5일/직전평균) | hist+gongsi_stock | 금호건설 x27, 대원전선 x2.7(CB+공매도과열) |
| PS5 | 일간 변동성(최대 |등락|) | hist+gongsi_stock | 져스텍 40%(단일계좌 상위), HPSP/원익IPS 상하한가 |
| PS6 | 신고가 돌파(마지막일 종가≥기간 전고) | hist+gongsi_stock | 정의 엄격(보수적) |
| PS7 | 기간 주목도 종합(|수익률|+2·최대일변+30·|거래대금증가율−1|) | rank+hist+gongsi_stock | 주목주 자동 랭크 + 공시 체인 |

**price-first 함정**: `rank`는 **날짜 파라미터가 없어 '오늘(최신 거래일)'만** 된다(PS1/PS2는 당일). `rank`
결과엔 ETF/ETN이 섞여 나와 `market_snapshot(exclude_etp=True)`로 제외. 공시 없는 종목은 `gongsi_jong`이
`success:false "No Data"`를 주는데 `call()`이 **빈 리스트로 처리**한다.

**전체종목 벌크(유니버스 상한 제거)** — 축·시장별로 갈린다(전부 실측):

| | KOSPI(m001) | KOSDAQ(m003) |
| --- | --- | --- |
| **현재일 전종목** | `basic_info_all_port`(100+필드) / `rank` | **`basic_info_all_port` 됨** — 숫자코드 1,771개 한 콜 성공 |
| **과거 기간 일별 OHLCV** | `hist_info_port(codelist, edate)` **날짜 루프**(저비용) | **없음** — hist_info_port가 404. 종목별 `hist_info` 루프뿐 |

- **코스콤 버그 + 우회(실측 확정)**: `basic_info_all_port`/`hist_info_port`/`hoga_info_port` codelist에
  **영숫자 6자리 코드**(스팩·최근상장: `0039P0` 등)가 **무따옴표로 1개라도 섞이면
  `Error while performing Query`로 배치 전체 실패**. → **각 코드를 작은따옴표로 감싸면**(`'0001A0'`)
  영숫자 코드까지 정상 조회된다(전 `*_port`에서 확인). 헬퍼(`_common.quote_codelist`)가 자동 적용하므로
  `numeric_only`는 기본 `False`(영숫자 종목 포함). 숫자코드만 원할 때만 `numeric_only=True`.
- `basic_info_all_port`는 **edate를 무시하고 항상 현재일**(과거일 불가). 과거 기간엔 못 쓴다.
- 헬퍼: `basic_all_port(codes, fam)` 현재일 벌크 · `hist_port(codes, edate)` KOSPI 과거일 벌크 ·
  `bulk_hist_period({'m001':[…],'m003':[…]}, s, e)` 코스피=날짜루프/코스닥=종목루프(`m003_limit`로 상위 N) ·
  `market_period_screen(s, e, m003_limit=N, min_amt_억=1)` 전종목 기간지표.
  실측: **KOSPI 1,129종목 3거래일 = ~6콜 1.1초**.
- 기간 스크리너(PS3~PS7)는 기본이 '거래대금 상위 유니버스'지만, KOSPI 전종목을 원하면
  `market_period_screen`으로 대체(코스닥 과거기간은 종목루프라 `m003_limit` 권장).

## 사용 예 (헬퍼)
```python
import news_gongsi_lab as lab
# 공시 이벤트 스터디 (시장 자동 라우팅) — want_gongsi로 그 종목의 공시 타임라인까지 붙임
lab.event_study("무상증자", "000500", "20260616")
lab.event_study("공매도과열", "058470", "20260612", want_short=True)
lab.event_study("최대주주변경", "293490", "20260619", want_loan=True, want_gongsi=True, want_news=True)
# 시세→공시 역방향
snap = lab.market_snapshot()                              # 오늘 전체 종목 스냅샷(rank, ETP 제외)
lab.screen_today(by="등락률", top=20, direction="top", snap=snap)
lab.explain_stock("002990", "20260601", "20260703")      # 급변 종목의 기간지표+공시목록
lab.notable_movers("20260601", "20260703", universe_top=150, top=15)
# 종목 뉴스(대형주 안전) / 감성 / 하루 전체 공시 마이닝
nj = lab.news_stock_chunked("005930", "20260601", "20260703")
rows = lab.gongsi_day("20260601", "20260630")            # 종목코드(NCD)로 이벤트 채굴
```
데모: `python news_gongsi_lab.py demo` · 러너: `python scenarios_price_first.py --only PS7` (등록 IP·샌드박스 밖).
