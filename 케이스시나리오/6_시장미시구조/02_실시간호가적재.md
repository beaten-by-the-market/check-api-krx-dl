# 시나리오 2. 실시간 호가 DB 적재 — KRX·NXT·통합 호가창 쌓기

## 한 줄 요약
KRX·넥스트레이드(NXT)·통합(둘의 최선호가) **세 거래소의 10단계 호가창**을 주기적으로 받아
**데이터베이스(PostgreSQL/TimescaleDB)에 실시간 적재**하는 파이프라인입니다. 앞 시나리오들이
"이미 있는 데이터를 분석"했다면, 이건 **분석할 데이터를 직접 쌓는 세팅**입니다.

## 이게 답해주는 질문 / 왜 하나
- 같은 종목이 KRX와 NXT에서 **가격·호가가 얼마나 다른가**(거래소 간 괴리)를 계속 기록하고 싶다
- 호가창의 **매수/매도 잔량 불균형**을 초 단위로 쌓아 단기 신호를 만들고 싶다
- 나중에 돌려볼 **나만의 호가 시계열 DB**가 필요하다 (API는 스냅샷만 주고 과거 호가는 안 줌)

> 핵심 동기: CHECK API는 **지금 이 순간의 호가 스냅샷**만 줍니다. 과거 호가 시계열은 제공되지
> 않으므로, 필요하면 **직접 폴링해서 DB에 쌓아 두는 수밖에** 없습니다. 이 시나리오가 그 방법입니다.

## 확인된 사실 (실측)
세 거래소 호가가 **모두 실제로 조회됩니다**(휴장일에도 마지막 스냅샷 반환). 예 — 삼성전자:

| 거래소 | 최우선 매수 | 최우선 매도 | 매수10단계잔량합 | 매도10단계잔량합 |
|---|---|---|---|---|
| KRX | 309,500 | 310,000 | 372,132 | 1,219,054 |
| **NXT** | 314,500 | 315,000 | 319,595 | 313,048 |
| 통합 | 314,500 | 315,000 | 319,595 | 313,048 |

→ KRX와 NXT 사이에 **약 1.6% 가격 괴리**가 실제로 관측됩니다(거래소별 마지막 체결 시각·유동성 차이).
바로 이런 걸 시계열로 쌓아 분석하려는 것이 이 시나리오의 목적입니다.

## LLM에게 이렇게 말하세요
- "KRX랑 NXT, 통합 호가를 DB에 실시간으로 쌓는 파이프라인 세팅해줘"
- "삼성전자·SK하이닉스 호가창 1초마다 받아서 TimescaleDB에 적재하는 스크립트 만들어줘"
- "거래소 간 호가 괴리를 계속 기록하고 싶어. DB 스키마랑 적재기 만들어줘"
- "지금 일요일이라 실데이터는 안 나오니까, 배선만 먼저 검증해줘"

## 무엇이 만들어지나 (산출물 3종)
1. **DB 스키마** (`hoga_schema.sql`) — 호가 스냅샷 테이블 `hoga_book`(하이퍼테이블) + 편의 뷰
   `hoga_l1`(스프레드·중간가·호가불균형 바로 계산). 자동 압축(7일 후)·파티셔닝 포함.
2. **적재 스크립트** (`hoga_ingest.py`) — 워치리스트 종목의 KRX·NXT·통합 호가를 주기 폴링해 적재.
   장 시간에만 돌고, 복수종목을 한 번에 받아 효율적.
3. **세팅 가이드** — DB 준비 → .env → 워치리스트 → 검증 → 실적재(장중) 순서.

## 이렇게 돌립니다
```bash
# 1) DB 한 번 준비
psql -d marketdata -f hoga_schema.sql

# 2) 배선 검증 — 네트워크·DB 없이 파서만 (일요일에도 OK)
python hoga_ingest.py --selftest

# 3) 실제 호가가 들어오는지 확인 (등록 IP, 적재는 안 함)
python hoga_ingest.py --venues krx,nxt,unified --once --dry-run --ignore-session

# 4) 장중 실적재 (평일 09:00~15:30 자동)
python hoga_ingest.py --watchlist watchlist.json --venues krx,nxt,unified --interval 1.0
```
> 오늘은 일요일이라 **4번(실적재)은 장중에 다시** 돌려야 하지만, **2·3번으로 배선은 지금 검증
> 가능**합니다(실제로 3번은 휴장인 지금도 마지막 스냅샷을 정상 반환하는 것까지 확인했습니다).

## 결과 읽는 법 / 쌓은 뒤 할 수 있는 것
- **거래소 간 괴리**: `hoga_l1` 뷰로 종목별 KRX vs NXT 중간가·스프레드 시계열을 뽑아 괴리 확대
  구간을 찾습니다.
- **호가 불균형**: `imbalance_l1`(=(매수1잔량-매도1잔량)/합)로 단기 매수/매도 압력을 봅니다.
- **유동성 벽**: 10단계 잔량합(ask_tot/bid_tot)의 증감으로 매물벽 형성·소멸을 추적합니다.
- **플래시 이벤트 근거**: [플래시 크래시](01_플래시크래시.md) 급변 순간의 호가창 붕괴를 이 DB로 확인합니다.

## 주의사항
- **"실시간" = 초 단위 폴링**입니다. CHECK API는 스트리밍(웹소켓)이 없어, 마이크로초 지연이 필요한
  초단타용이 아니라 **초 단위 기록·분석용**입니다. 더 빠른 게 필요하면 거래소 직결 피드가 답입니다.
- **등록 IP PC에서만** 실데이터가 들어옵니다(그 외 환경은 인증 거부).
- **NXT는 연장거래시간**이 있어 스냅샷 시각이 KRX와 다를 수 있습니다 → 괴리가 "시간차"일 수 있으니
  수집시각과 함께 해석하세요.
- **저장량**: 종목 100 × 3거래소 × 1초면 하루 수백만 행입니다. 관심종목을 좁히거나, 간격을 늘리거나,
  "직전과 같으면 건너뛰기"로 크게 줄일 수 있습니다(가이드에 방법 포함).
