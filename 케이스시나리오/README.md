# 케이스 시나리오 — 한국어로 말하면 나오는 자본시장 분석 모음

이 폴더는 KOSCOM CHECK API 위에서 **"이런 걸 보고 싶어"라고 한국어로 말하면 LLM이 알아서
데이터를 가져와 표·해석을 만들어 주는" 분석 시나리오**들을, **도메인(자본시장) 관점**에서
유형별로 정리한 것입니다.

코드도, API도, LLM 사용법도 몰라도 됩니다. 각 문서는 *무엇을 말하면 무엇이 나오는지*를
**실제 예시(전부 실데이터로 검증, 2026년 6~7월 기준)**와 함께 보여줍니다.

## 어떻게 쓰나요?
1. 아래 표에서 하고 싶은 분석의 **유형**을 고릅니다.
2. 그 폴더의 `00_README.md`를 열어 세부 시나리오를 고릅니다.
3. 각 문서의 **"LLM에게 이렇게 말하세요"** 예시 문장을 (종목·기간만 바꿔) 채팅에 입력합니다.
4. 나오는 결과를 **"결과 읽는 법"** 설명과 함께 해석합니다.

> 말투는 자유입니다. "삼성전자 작년 실적 어땠어?"처럼 편하게 물어도 되고, 예시 문장을
> 복사해 종목만 바꿔도 됩니다. 용어를 정확히 몰라도 LLM이 되묻습니다.

## 공통으로 알아둘 점 (딱 4가지)
- **실데이터는 등록 IP PC에서만**: 이 분석들이 실제 숫자를 뽑으려면 KOSCOM에 등록된 IP의
  PC에서 Claude Code로 돌려야 합니다(자세히는 리포 루트 `SETUP.md`). 그 외 환경에선 "방법·예시"까지만.
- **시장 구분**: 코스피(유가증권)와 코스닥은 데이터 창구가 다릅니다. LLM이 알아서 맞춰주지만,
  결과에 "시장: 코스피/코스닥" 표기가 붙는 이유입니다.
- **거래소 3창구(KRX·NXT·통합)**: 2025년 넥스트레이드(NXT) 출범으로 같은 종목이 KRX·NXT 두 곳에서
  거래됩니다. **거래대금/거래량/순위는 KRX만 보면 실제보다 30~45% 적습니다**(삼성전자는 거래의 약
  절반이 NXT). 그래서 시세 스크리너·장중 분석은 **통합(KRX+NXT) 기본**으로 바뀌었고, "KRX 공식으로",
  "NXT만"이라고 하면 바꿀 수 있습니다. 단 공매도·대차·수급·복수종목 기간시세는 통합에 API가 없어
  KRX 기준이며, 종가·등락률 "공식값"도 KRX입니다.
- **날짜 기준**: 예시들은 2026년 6~7월 데이터입니다. "지난주", "어제"처럼 말해도 되고,
  `20260703`처럼 날짜를 직접 줘도 됩니다.
