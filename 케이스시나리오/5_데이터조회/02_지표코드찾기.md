# 시나리오 2. 지표·코드 찾기 — "이 지표/코드 뭐야?"를 먼저 확인

## 한 줄 요약
조회하기 전에 **"그 지표를 어디서, 어떤 코드로 보는지"**를 먼저 찾아 주는 시나리오입니다.
지표명이 제목에 없어도 데이터 항목(필드)까지 뒤져 창구를 찾아냅니다.

## 이게 답해주는 질문
- 이 지표가 이 데이터에 **아예 있긴 있나?**
- 원/달러 환율, 코스피 종합 지수는 **어떤 코드**로 봐야 하나?
- 베이시스·신용융자 같은 건 어느 창구에서 나오나?
- 삼성전자 종목코드가 뭐지?

바로 [시나리오 1](01_아무데이터나조회.md)에서 뽑기 전에, **"먼저 어디서 보는지"를 확인**하는 준비 단계입니다.

## LLM에게 이렇게 말하세요
- "베이시스 데이터 있어?"
- "원/달러 환율은 어떤 코드로 봐?"
- "코스피 종합 지수 코드 뭐야?"
- "신용융자 잔고 어디서 봐?"
- "삼성전자 종목코드 알려줘"

## 그러면 이런 결과가 나옵니다
**검증된 앵커 예시:**

| 지표 | 어디서 / 코드 |
|---|---|
| 원/달러 환율 | 코드 `00USDSP` |
| 코스피 종합 지수 | 코드 `1` |
| 삼성전자 | 종목코드 005930 |
| 베이시스·공매도 과열·신용융자 등 | 제목엔 없어도 **데이터 항목(필드)**에 있어 찾아줌 |

핵심은, **지표명이 목록 제목에 안 보여도 없는 게 아니라는 점**입니다. 예컨대 "베이시스"는
독립 메뉴 이름이 아니라 파생 데이터의 한 **필드**로 들어 있어, 필드까지 뒤져 찾아냅니다.

## "검색이 비어도 없는 게 아니다"
지표명으로 한 번 검색해서 안 나온다고 바로 "없음"이라고 판정하지 않습니다. LLM은
**코드 카탈로그와 개별 항목(필드)까지** 뒤져 다시 찾습니다.

- 실제로 개발 과정에서 **지수·원/달러 환율·해외 금리·유가**를 처음엔 "없다"고 했다가,
  뒤에 코드·필드를 더 뒤져 찾아낸 사례가 있습니다.
- 그래서 이 데이터에서 **"없음"은 잠정 판정**입니다. 한 번 못 찾아도, 다른 이름·다른 창구로
  다시 확인한 뒤에야 최종적으로 "범위 밖"이라고 말합니다.

## 진짜 범위 밖 (솔직히 안내)
아래는 여러 창구를 뒤져도 이 데이터에서 나오지 않아, **범위 밖이라고 명확히 안내**하는 항목들입니다.

- **VI(변동성완화장치) 발동 내역**
- **VKOSPI(변동성지수)**
- **투자자예탁금**
- **배당락 / 권리락** 이벤트
- **국내주식 배당수익률 / PER** (해외 종목은 있으나 국내는 없음)
- **Brent / 두바이 유가**
- **투자주체별 공매도** (공매도 총량은 있어도 주체별 분해는 없음)

## 결과 읽는 법
- **애매한 지표는 조회 전에 이 시나리오로 먼저 확인하세요.** "이거 있어?"라고 물으면
  LLM이 "이건 이 창구/이 코드로 본다", "이건 범위 밖이다"로 정리해 줍니다.
- **"없다"고 나와도 한 번 더 밀어볼 수 있습니다.** 위처럼 "없음"은 잠정 판정이라, "다른 이름으로
  다시 찾아봐"라고 하면 코드·필드를 더 뒤져 봅니다.
- 창구·코드를 확인했으면, 실제 값은 [시나리오 1(아무 데이터나 조회)](01_아무데이터나조회.md)에서 뽑습니다.

## 이렇게도 바꿔 말할 수 있어요
- **코드 → 값 바로**: "원/달러 환율 코드로 최근 값까지 같이 뽑아줘"
- **여러 지표 한 번에**: "베이시스랑 선물 미결제 둘 다 어디서 보는지 알려줘"
- **범위 확인만**: "국내 배당수익률 데이터 있어? 없으면 없다고 말해줘"
- **종목코드 일괄**: "반도체 대표주들 종목코드 표로 정리해줘"
